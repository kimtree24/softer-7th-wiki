## 4. 오늘의 학습

### 4.1 미션 수행
#### (1) M3
multiprocessing.Queue는 자료구조가 아니라 프로세스 간 통신(IPC)을 위한 통로

프로세스는 메모리를 공유하지 않음. 따라서 list, deque 같은 일반 자료구조는 다른 프로세스에서 접근할 수 없음

이를 해결하기 위해 multiprocessing.Queue가 존재함.

#### (2) M4 - Queue.get_nowait() & Exception Handling 정리

 1)) Queue.get_nowait()는 큐에서 아이템을 즉시(non-blocking) 가져오는 메서드
```
	•	아이템이 있으면 → 즉시 반환
	•	아이템이 없으면 → 기다리지 않고 예외(Empty) 발생
```

즉, get()과 달리 대기(blocking)하지 않는 방식이다.

2)) get() vs get_nowait() 차이

```
- get()
	기본 동작 : blocking
	큐가 비었을 때 : 아이템이 올 때까지 대기
	종료 처리 : sentinel 등 별도 설계 필요
	
- get_nowait()
	기본 동작 : non-blocking
	큐가 비었을 때 : 즉시 예외 발생
	종료 처리 : Empty 예외로 종료 판단 가능
```


3)) 예외 타입: Empty

get_nowait()에서 큐가 비어 있으면 Empty 예외가 발생한다.

일반적으로 다음과 같이 import해서 사용한다.

```
from queue import Empty
```

4)) multiprocessing 작업 분배에서 get_nowait()가 좋은 이유

multiprocessing.Queue를 작업 큐로 쓰면 여러 워커가 동시에 다음 코드를 실행

```
task = tasks_to_accomplish.get_nowait()
```

	•	작업이 있으면 → 하나 가져가 처리
	•	작업이 없으면 → Empty → 워커 종료

결과적으로 워커 수와 상관없이 작업이 자연스럽게 분산 처리
