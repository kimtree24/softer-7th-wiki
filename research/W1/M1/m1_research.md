# 데이터셋 컬럼 정의 (mtcars)

## 컬럼 설명

| 컬럼명 | 의미 | 단위 | 해석 포인트 |
|------|------|------|------------|
| mpg | 연비 (Miles per US gallon) | mile / gallon | 값이 클수록 연비가 좋음 |
| cyl | 실린더 수 (Number of cylinders) | 개 | 엔진 크기, 출력과 관련 |
| disp | 배기량 (Displacement) | cubic inch (cu.in.) | 클수록 엔진 규모가 큼 |
| hp | 마력 (Gross horsepower) | hp | 차량 출력 성능 |
| drat | 후륜 차축 기어비 (Rear axle ratio) | 비율 | 가속력, 연비 특성에 영향 |
| wt | 차량 무게 (Weight) | 1000 lb 단위 | `wt=3` → 3000 lb |
| qsec | 1/4 마일 주행 시간 | 초 | 작을수록 가속 성능이 좋음 |
| vs | 엔진 형태 (V/S) | 0 / 1 | 0 = V형 엔진, 1 = 직렬 엔진 |
| am | 변속기 종류 (Transmission) | 0 / 1 | 0 = 자동, 1 = 수동 |
| gear | 전진 기어 수 | 개 | 변속 단계 수 |
| carb | 카뷰레터 수 | 개 | 연료 공급 장치 수 |

## 참고 사항

- `vs`, `am` 컬럼은 숫자형이지만 범주형 변수
- `mpg`는 연비 분석에서 주요 타깃 변수로 자주 사용된다.
- `wt`, `disp`, `hp`는 연비(`mpg`)와 음의 상관관계를 가지는 경우가 많다.

# Pandas DataFrame methods

## 1. Pandas DataFrame 개요

**설명**

`DataFrame`은 **행(row)과 열(column)** 로 이루어진 표 형태의 데이터 구조

CSV, Excel, SQL 결과 등을 다룰 때 사용.

**사용 예시**
```
import pandas as pd

df = pd.read_csv("data.csv")
```

## 2. DataFrame 기본 메서드 & 속성

### 2.1 head

**설명**

head()는 DataFrame의 상위 n개 행을 반환.
기본값은 5행.

**사용 예시**
```
df.head()
df.head(10)
```

**활용**
- 데이터가 정상적으로 로드되었는지 확인
- 컬럼명과 값의 형태를 빠르게 파악

### 2.2 tail

**설명**
tail()은 DataFrame의 하위 n개 행을 반환.

기본값은 5행.

**사용 예시**
```
df.tail()
df.tail(10)
```

**활용**
- 정렬 여부 확인
- 시계열·로그 데이터의 마지막 상태 확인

### 2.3 shape

**설명**
shape는 DataFrame의 차원 정보(행 수, 열 수) 를 반환.

**사용 예시**
```
df.shape
```

**출력 형식**
```
(행 개수, 열 개수)
```

**활용**
- 전체 데이터 규모 파악
- 전처리 전·후 데이터 변화 확인

### 2.4 columns

**설명**

columns는 DataFrame의 컬럼 이름 목록을 반환.

**사용 예시**
```
df.columns
```

**활용**
- 컬럼 구조 파악
- 컬럼 선택 및 이름 변경 시 참고

### 2.5 info

**설명**

info()는 DataFrame의 전체 구조 요약 정보를 출력

**사용 예시**
```
df.info()
```

**제공 정보**
- 행 개수
- 컬럼 이름
- 데이터 타입(dtype)
- 결측치(null) 개수
- 메모리 사용량

**활용**
- 데이터 타입 오류 탐지
- 결측치 존재 여부 확인

### 2.6 dtypes

**설명**
dtypes는 각 컬럼의 데이터 타입을 반환.

**사용 예시**
```
df.dtypes
```

**활용**
- 숫자형 / 문자형 컬럼 구분
- 타입 변환 필요 여부 판단

### 2.7 describe

**설명**

describe()는 숫자형 컬럼에 대한 통계 요약 정보를 반환.

**사용 예시**
```
df.describe()
```

**기본 제공 통계**
- count
- mean
- std
- min
- 25%, 50%(median), 75%
- max

**문자형 컬럼까지 포함**
```
df.describe(include="all")
```

**활용**
- 데이터 분포 파악
- 이상치(outlier) 탐지

### 2.8 rename

**설명**
- DataFrame의 행(index) 또는 열(column)의 이름을 변경하는 메서드.
- 원본을 그대로 두고 새 DataFrame을 반환.

**사용 예시**

(1) 컬럼 이름 변경
```
df = df.rename(columns={"old_name": "new_name"})
```
(2) 인덱스 이름 변경

1. 특정 인덱스 값 변경
    ```
    df = df.rename(index={0: "first_row", 1: "second_row"})
    ```

2. 인덱스 이름 자체 변경
    ```
    df.index.name = "user_id"
    ```

**inplace=True 옵션**
```
df.rename(columns={"a": "A"}, inplace=True)
```
- 반환값 없음(None)
- 원본 DataFrame 직접 변경
`권장하지 않음 -> (디버깅, 가독성 때문)`

### 2.9 unique

**설명**
컬럼에 존재하는 고유한 값들의 배열을 봔환
해당 컬럼에 어떤 값들이 있는지 확인할 때 사용

**사용 예시**
```
df['gear'].unique()
```

**경우의 수**
```
df['gear'].nunique()
```

### 2.10 groupby

**설명**
하나 이상의 컬럼을 기준으로 데이터를 그룹화하는 메서드
특정 기준으로 묶어서 통계를 내고 싶을 때 사용

**사용 예시**
```
df.groupby(['gear', 'am']).size()
```

**집계 메서드**
- size() : NaN 포함
- count() : NaN 제외
- sum()
- mean()
- min(), max()
- median(), std()
- filter()

## 3. matplotlib

### 3.1 히스토그램

**설명**
히스토그램은 **각 변수의 분포(distribution)** 를 확인하기 위한 가장 기본적인 시각화 도구
EDA에서 가장 먼저 수행해야 하는 과정

**히스토그램 그리는 이유**
1) 데이터 분포 형태 파악
- 값이 특정 구간에 몰려 있는지
- 정규분포에 가까운지
- 한쪽으로 치우친(skewed) 분포인지 확인

2) 이상치(outlier) 탐지
- 극단적으로 크거나 작은 값이 존재하는지
- 데이터 품질 문제 또는 실제 의미 있는 값인지 판단

3) 변수 스케일 및 변환 필요성 판단
- 로그 변환, 표준화 등이 필요한 변수인지 사전에 판단 가능
- 모델링 전 전처리 방향을 결정하는 데 중요한 기준

4) 변수 간 비교의 기초 자료
- 이후 scatter plot이나 상관계수 해석 시 “이 변수는 어떤 분포를 가진다”라는 사전 이해를 제공.

### 3.2 Scatter 차트

**설명**
두 수치형 변수 간의 관계를 시각적으로 확인하기 위한 그래프

**Scatter 차트 그리는 이유**
1) 변수 간 관계(상관성) 확인
- 한 변수가 증가할 때 다른 변수가 함께 증가하는지(양의 관계)
- 반대로 감소하는지(음의 관계)
- 관계가 거의 없는지 확인

2) 선형 관계 여부 판단
- 상관계수는 선형 관계만 측정하므로, 실제로 선형 패턴이 존재하는지 산점도로 먼저 확인하는 것이 중요

3) 군집(clustering) 및 패턴 발견
- 특정 범주(예: 실린더 수)에 따라 점들이 묶이는 현상
- 데이터가 자연스럽게 여러 그룹으로 나뉘는지 확인 가능

4) 이상치 시각적 확인
- 전체 패턴에서 벗어난 점들을 직관적으로 파악

## 4. 상관 관계

### 4.1 상관 계수
두 변수 간의 선형 관계의 강도와 방향을 수치로 표현한 값


### 4.2 상관계수 값의 의미
- 값의 범위: `-1` ~ `1`
- `1`에 가까울수록: 강한 양의 상관관계
- `-1`에 가까울수록: 강한 음의 상관관계
- `0`에 가까울수록: 선형 관계 거의 없음

일반적인 해석 기준:
- |corr| ≥ 0.7 : 강한 상관관계
- 0.4 ≤ |corr| < 0.7 : 중간 상관관계
- |corr| < 0.4 : 약한 상관관계

### 4.3 양(+) / 음(-) 상관관계의 의미
- 양의 상관관계: 한 변수가 증가하면 다른 변수도 증가하는 경향
- 음의 상관관계: 한 변수가 증가하면 다른 변수는 감소하는 경향

예:
- `mpg`와 `wt`의 강한 음의 상관관계  
  → 차량이 무거울수록 연비가 나빠지는 경향

### 4.4 상관계수의 활용 목적
- 변수 간 관계를 한눈에 요약
- 중복 정보(다중공선성) 탐지
- 모델링 시 feature selection의 근거 제공

### 4.5 주의 사항
- 상관관계는 **인과관계 아님**
- 두 변수가 함께 변한다고 해서 하나가 다른 하나의 원인이라고 단정할 수는 없음.

상관계수는 **scatter plot과 함께 해석해야 의미가 정확해짐**.
