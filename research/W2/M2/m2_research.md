# **Python multiprocessing.Process 정리**

## **1. Process 클래스란?**

하나의 함수를 독립된 프로세스에서 실행하기 위해 직접 프로세스를 생성하는 클래스 

- 각 Process는 **완전히 독립된 메모리 공간**을 가진다.
- 실행 순서, 시작 시점은 **운영체제(OS) 스케줄러**가 결정한다.
- 개발자가 프로세스의 생명주기(start / join)를 **직접 제어**한다.

## **2. 왜 Process를 써야 하는가?**

- 하나의 함수를 **여러 프로세스에서 실행**할 수 있는지
- 함수에 **인자를 전달**할 수 있는지
- 기본 인자(default argument) 처리
- 프로세스의 시작과 종료를 **명시적으로 관리**할 수 있는지
- 멀티프로세스에서는 **실행 순서를 가정하면 안 된다는 점**

---

## **3. 기본 구조**

```python
from multiprocessing import Process

def task():
    pass

if __name__ == "__main__":
    p = Process(target=task)
    p.start()
    p.join()
```

### **역할 분리**

- **메인 프로세스**
    - Process 생성
    - start / join 호출
- **자식 프로세스**
    - target 함수 실행

## **4. 실행 순서가 매번 달라지는 이유**

`p.start()` 의 의미는

“지금 실행해라” X

“실행 가능 상태로 만들어 달라” O

- 실제 실행 순서는 **OS 스케줄러**가 결정
- CPU 상태, 코어 수, 부하에 따라 매번 달라짐
- join()은 **종료 대기만 보장**, 순서 보장 X

멀티프로세스 환경에서는 **출력 순서를 가정하면 안 된다**

## **5. Process vs Pool 핵심 차이**

| **항목** | **Process** | **Pool** |
| --- | --- | --- |
| 프로세스 생성 | 수동 | 자동 |
| 작업 분배 | 직접 | 자동 |
| 제어 수준 | 높음 | 낮음 |
| 사용 목적 | 개별 제어 | 대량 동일 작업 |

Process는 **저수준 제어** 

Pool은 **고수준 병렬 처리 API**
