1. Docker를 사용하는 이유

1) 실행 환경의 일관성 보장
- Docker는 애플리케이션 실행에 필요한 OS 환경, 라이브러리, 패키지 버전, 설정 파일을 이미지 단위로 묶어 제공한다.
- 로컬에서는 되는데 서버에서는 안 되는 문제를 줄일 수 있다.

2) 배포 단순화
- Docker 사용 시 이미지를 한 번 만들고 어디서든 docker run만 하면 동일한 환경 실행 가능
- 배포 과정이 명확하고 재현 가능해진다.

3) 인프라 독립성
- Docker 이미지는 로컬 PC, AWS EC2, 다른 클라우드 환경 어디서든 동일하게 실행 가능하다.

2. Docker의 불편한 점

1) 초기 학습 비용
- Dockerfile 작성, 이미지 / 컨테이너 개념, 볼륨, 포트, 네트워크 이해 등 초기 러닝커브가 존재한다.

2) 디버깅의 어려움
- 컨테이너 내부 로그를 직접 확인해야 함

3. 여러 EC2에 여러 Docker 컨테이너를 배포하려면?

(1) 컨테이너 오케스트레이션 도입

1) Kubernetes (EKS)
- 여러 EC2 노드를 하나의 클러스터로 묶어 관리

2) AWS ECS (Elastic Container Service)
- AWS에 최적화된 컨테이너 오케스트레이션
- EC2 기반 또는 Fargate(Serverless) 기반 선택 가능
